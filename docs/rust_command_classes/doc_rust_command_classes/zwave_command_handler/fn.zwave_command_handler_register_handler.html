@brief Register a command class handler.

This API replaces the legacy REGISTER_HANDLER() macro. Command handlers should perform their own initialization and register themselves to the command handler framework.

There is no sequence/order requirement, meaning that this function can be invoked before the Command Class handler framework is initialized.

@param new_command_class_handler A struct that contains the zwave_command_handler_t info that will be used for registering a Command Class handler.

Command class handlers can be fully disabled by setting both support and control to false. @returns SL_STATUS_OK If the Command Class handler was successfully registered @returns SL_STATUS_FAIL Otherwise
