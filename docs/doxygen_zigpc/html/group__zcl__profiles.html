Detailed Description

The role of the ZigPC ZCL profiles component is to contain the definitions and procedures to help translate ZCL Cluster information recevied from Unify and from the Zigbee network.

Populate the given array with supported command names based on the cluster name passed in. If the cluster name passed in is not supported, nothing will be populated.

Parameters
cluster_nameCluster name character array.
dest_command_listDestination list of char array references.
dest_command_countCapacity of destination array passed in.
commands_populatedPointer to location to store count of commands populated.
Returns
sl_status_t SL_STATUS_OK on successful population of destination list, SL_STATUS_NULL_POINTER if invalid pointers are sent, SL_STATUS_EMPTY if the cluster name or the destination list has zero capacity, or SL_STATUS_WOULD_OVERFLOW if the destination list has insufficient size.
