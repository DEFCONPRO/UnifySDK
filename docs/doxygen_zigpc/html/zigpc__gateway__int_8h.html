◆ zigpc_gateway_load_device_table_info()

sl_status_t zigpc_gateway_load_device_table_info ( void  )

Retrieve persisted information related to Z3Gateway and ZigPC Gateway from the datastore. This includes:

  • GSDK device-table plugin state: first the number of entries stored in the datastore is retrieved. Then, each datastore entry (and the associated cluster list) is loaded and imported using the z3gateway device table import entry API.
Returns
sl_status_t SL_STATUS_OK if all information was loaded from datastore successfully, error status if not.

◆ zigpc_gateway_persist_device_table_info()

sl_status_t zigpc_gateway_persist_device_table_info ( void  )

Persist information related to Z3Gateway and ZigPC Gateway to the datastore. This includes:

  • GSDK device-table plugin state: first the number of entries used by the device table is retrieved. Then, each entry (and the associated cluster list) is exported using the z3gateway API and saved in the datastore as three separate info types: device_table size, device_table_entry[i], and device_table_entry[i].clusterList.
Returns
sl_status_t SL_STATUS_OK if all information was store to datastore successfully, error status if not.
