Detailed Description

This component is responsible transmission of application frames.

This component handles all the queuing logic regarding Z-Wave frames transmission, back-off, and some frame encapsulation such as Security 0, Security 2, Transport service, Multi Channel (endpoints) and CRC16 encapsulation.

The component features a prioritized queue by QoS. Frames with the highest QoS are transmitted first. There is no guarantee over the order of elements with identical QoS. The low-level encapsulation control frames, such as nonce report etc, must be given the highest priority.

The external interface is described in Z-Wave TX interface.

There are 4 internal sub-modules:

  • Z-Wave TX callbacks is a list of callbacks registered to other components.
  • Z-Wave TX Process takes care of processing the queue and interfacing with Z-Wave API.
  • Z-Wave TX Intercept intercepts the Z-Wave Frame that can not be sent using zwapi_send_data command and send the frame using other zwapi transport commands.
  • Z-Wave TX queue is the data model definition for the Z-Wave TX queue ands its elements.
