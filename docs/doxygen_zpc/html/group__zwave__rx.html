Detailed Description

Component responsible for initializing the Z-Wave API, listening to incoming frames, and dispatching them.

This component is composed of 2 sub-parts:

  1. Z-Wave RX fixture : This part is responsible for the Z-Wave API initialization using the Configuration Extension for parameters.
  2. Z-Wave RX process : This component is responsible for continuously polling incoming messages from the Z-Wave module using a The Contiki RTOS process.

In case of received Z-Wave frames, they will be passed on to the (FIXME: UIC-95) component handling de-encapsulation of incoming Z-Wave frames (S0, S2, Transport Service, Multi Channel, CRC16, Multi Command)

For some other events, such as received NIF, SmartStart inclusion or node deleted from the network, the Z-Wave Controller will be notified.
